# PHASE 2C — 移动端 Header 结构修复

日期：2026-04-19
负责人：Codex

## 问题
- 移动端菜单出现遮挡与内容看不全。
- 原布局在小屏下可滚动性不足，信息层级偏乱。
- 顶栏与菜单层发生双层叠加，出现视觉重复（如 Logo 重影）。

## 本次修复
文件：`src/components/layout/Header.tsx`

1. 菜单容器改为全屏抽屉层
- 菜单改为“从固定顶栏下方展开”（`top-16`），避免与顶栏重复叠层。
- 顶栏仅保留一层，菜单关闭/打开都用同一个右上角按钮切换。

2. 内容区改为可滚动
- 菜单主体改为 `flex-1 + overflow-y-auto`。
- 解决小屏设备下底部内容被裁切、不可达的问题。

3. 信息架构简化（仿大厂常见模式）
- 分组清晰：`Main` 与 `Seedance`。
- 链接样式改为统一卡片化行项，降低视觉噪声。
- Seedance 分组保留 3 个核心入口，减少无关跳转。

4. 账户区与语言区重排
- 账户卡片放入滚动内容中，避免被固定底部挤压。
- 语言切换器保留在主体内容区，保证可见与可操作。

5. 底部版权区固定在抽屉底部
- 与菜单主体分离，维持一致层级与可读性。
- 版权保留在滚动容器内，避免极端视口下被裁切。

## 验收点
- iPhone 常见视口下（390x844 / 393x852）可完整滚动查看全部菜单项。
- 打开菜单后，无顶部遮挡与点击穿透。
- 用户可在首屏快速识别主路径：主页、生成器、Seedance 三页。
